## 2. Context

`v0.2.0` established the AgentOS skeleton: a FastAPI-based `AgentOS` instance, the Team Leader Agent, PostgreSQL-backed session/chat-history storage, and two built-in tools. `v0.2.1` is the first "runnable surface" milestone: it mounts the **AG-UI** interface on AgentOS so a client can send a message and receive a streaming response.

Key constraints:

- Use Agno's first-party `AGUI` interface (`agno.os.interfaces.agui.AGUI`).
- Keep the scope minimal: one agent, one endpoint, one manual test.
- Do not build a custom frontend; use `curl` for the manual test.
- Tests must not require a live LLM or a running database.

---

## 3. Scope

### 3.1 In Scope

1. **AG-UI dependency**
   - Add `agno[agui]` to `pyproject.toml` so the `AGUI` interface and `ag-ui-protocol` types are available.
   - Regenerate `uv.lock`.

2. **Wire AGUI into AgentOS**
   - Import `AGUI` in `src/spectres/main.py`.
   - Pass `interfaces=[AGUI(agent=team_leader_agent)]` to `AgentOS`.
   - Keep the existing Team Leader Agent and database wiring unchanged.

3. **Runnable server entry point**
   - Implement an `if __name__ == "__main__":` block in `src/spectres/main.py` that calls `agent_os.serve(app="spectres.main:app")`.
   - Start AgentOS with:
     ```bash
     uv run python -m spectres.main
     ```
   - The host and port are controlled by optional `AGENT_OS_HOST`/`AGENT_OS_PORT` environment variables (default: `localhost:7777`).

4. **Environment variables**
   - Add optional `AGENT_OS_HOST` and `AGENT_OS_PORT` documentation to `.env.example` if they influence the serve command.
   - Keep database and Team Leader LLM variables already defined in `v0.2.0`.

5. **Automated tests**
   - Use FastAPI `TestClient` to test the mounted endpoints without a live server:
     - `GET /status` returns HTTP 200.
     - `POST /agui` exists and accepts a minimal `RunAgentInput`-style JSON payload.
   - Mock or patch the agent run so no LLM API key or database is required.

6. **Simplest manual test**
   - Start the local PostgreSQL container (`docker compose up -d`).
   - Start AgentOS with the documented serve command.
   - Run:
     ```bash
     curl http://localhost:7777/status
     ```
   - Run:
     ```bash
     curl -N -X POST http://localhost:7777/agui \
       -H "Content-Type: application/json" \
       -d '{
         "thread_id": "test-thread",
         "run_id": "test-run",
         "state": {},
         "messages": [{"id": "msg-1", "role": "user", "content": "你好"}],
         "tools": [],
         "context": [],
         "forwarded_props": {}
       }'
     ```
   - Verify that `/status` responds and `/agui` streams an AG-UI event response.

7. **Documentation**
   - Update `README.md` with:
     - What AG-UI support means for this milestone.
     - How to start AgentOS.
     - The exact curl commands for the manual test.

### 3.2 Out of Scope

- Custom frontend (Dojo, React, or otherwise).
- Authentication, authorization, or multi-tenancy on the AG-UI endpoint.
- Production deployment manifests or Docker image for Runtime.
- Streaming content assertions beyond a basic non-empty response.
- Multi-agent routing / slave agent invocation.
- Long-term memory or knowledge base.
