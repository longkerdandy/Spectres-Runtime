# Milestone Plan — v0.1.0

> Establish the development environment, IDE support, code quality workflow, and GitHub security scanning for Spectres Runtime.
> No framework-specific code or application modules are implemented.
>
> Status: `completed`

## 2. Context

Spectres Runtime is a Python-based AI agent runtime. Before any application code is written, the repository needs a reproducible local development environment, IDE configuration, code quality automation, and baseline security scanning on GitHub.

Prior dependencies: none.

---

## 3. Scope

### 3.1 In Scope

1. **Python development environment**
   - `pyproject.toml` defining the project and Python version constraint.
   - Dependency groups for runtime and development.
   - Virtual environment setup documented.
   - A script or command to install the project in editable mode.

2. **VSCode support**
   - `.vscode/extensions.json` recommending essential extensions.
   - `.vscode/settings.json` configuring format on save, default formatter, and linting integration.
   - `.vscode/launch.json` with a basic debug configuration for Python.

3. **Code quality management**
   - Linting and formatting configured in `pyproject.toml`.
   - Type checking configured.
   - Test runner configured with asyncio support.
   - A `Makefile` or `scripts/` commands for `test`, `lint`, `format`, and `typecheck`.

4. **GitHub Actions**
   - `.github/workflows/ci.yml` running lint, typecheck, and tests on pull requests and pushes to the main branch.
   - Matrix strategy across supported Python versions if applicable.

5. **GitHub Code Scanning & Dependency Security**
   - `.github/workflows/codeql.yml` enabling CodeQL analysis for Python.
   - Dependency review workflow for pull requests.
   - Dependabot configuration for automated dependency update alerts.

### 3.2 Out of Scope

- Application framework setup (Agno, AgentOS)
- Source code modules (`src/spectres/`, agents, tools, memory, etc.)
- Configuration system for runtime behavior
- Logging framework setup
- REST/WebSocket/API endpoints
- Client or Edge protocol implementations
- Docker or deployment manifests
- Business logic or task-specific code

## 4. Success Criteria

- [x] `pyproject.toml` exists and defines the project, Python version, and dependency groups.
- [x] A virtual environment can be created and dependencies installed by following `README.md`.
- [x] `.vscode/` files provide a consistent editor experience.
- [x] Linting command runs and reports zero issues on the initial codebase.
- [x] Formatting command runs and leaves the codebase clean.
- [x] Type checking command runs with no errors.
- [x] Test command runs successfully, even if no tests exist yet.
- [x] CI workflow runs lint, typecheck, and tests on GitHub.
- [x] CodeQL workflow runs Python security analysis on GitHub.
- [x] Dependency review workflow runs on pull requests.
- [x] Dependabot is configured for dependency alerts.
- [x] No secrets or generated artifacts are committed.
