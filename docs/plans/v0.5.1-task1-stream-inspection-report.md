## 1. Method

1. Started the Runtime locally with `./scripts/runtime_start.sh`.
2. Confirmed the recipe agent is registered at `GET /agents`.
3. Sent a streaming request to `POST /agents/recipe/runs` with `stream=true`.
4. Recorded all Server-Sent Events to a JSONL file.
5. Analyzed event types, keys, and field shapes with a small Python script.
6. Inspected the built `Agent` object directly to verify tool registration and
   reasoning configuration.

**Important:** the AgentOS `runs` endpoint expects `application/x-www-form-urlencoded`
form data, not JSON. Sending JSON returns HTTP 422 with a missing `message` error.

## 2. Event Type Summary

For a single turn ("帮我安排一个四菜一汤的晚餐"), the stream emitted 465 events:

| Event | Count | Purpose |
|---|---|---|
| `RunStarted` | 1 | Run lifecycle start. |
| `ModelRequestStarted` | 1 | Model call starts. |
| `RunContent` | 460 | Token deltas. Each event carries either `reasoning_content` or `content`. |
| `ModelRequestCompleted` | 1 | Model call finished; contains token usage. |
| `RunContentCompleted` | 1 | Content generation finished. |
| `RunCompleted` | 1 | Final payload with full content, reasoning, and metrics. |

**Notably absent:** `ToolCallStarted`, `ToolCallCompleted`, `ReasoningStarted`,
`ReasoningStep`, `ReasoningCompleted`. The current recipe agent did not invoke
any tools.

Observations:

## 4. Critical Findings

### 4.1 Tools are not registered on the agent

Direct inspection of the built agent shows:

```python
agent = build_recipe_agent(settings)
print(agent.tools)        # []  (empty)
print(agent.knowledge)    # None
print(agent.reasoning)    # False
```

The current `build_recipe_agent()` constructs an `Agent` without the recipe
search/detail tools and without `ReasoningTools`. The default instructions still
say "Search the recipe knowledge base", so the model hallucinates retrieval
instead of calling tools.

This is a regression from the v0.4.1 tool-driven refactor intent. Until fixed,
no `ToolCallStarted` / `ToolCallCompleted` events will appear in the stream.

### 4.2 Reasoning is model-native, not structured steps

The model emits reasoning tokens, which Agno surfaces as `reasoning_content`
(string deltas). There are **no** `ReasoningStep` objects with titles, results,
or confidence. If the Web plan needs structured reasoning steps, the Runtime
must either:

1. Use `ReasoningTools` so the model emits explicit `think()` / `analyze()`
   tool calls, which AgentOS renders as normal tool-call events, or
2. Update the Web plan to render the flat `reasoning_content` string as a
   single expandable reasoning block.

### 4.3 Token metrics are already sufficient

No Runtime work is needed for §2 of the Web plan (token usage display). Both
streaming and non-streaming responses already carry:

- `input_tokens`
- `output_tokens`
- `total_tokens`
- `duration` / `time_to_first_token`

## 5. Mapping to Spectres-Web v0.5.1 Requirements

| Web plan requirement | Current Runtime state | Gap |
|---|---|---|
| `message.extra_data.reasoning_steps` with title/result/confidence | `reasoning_content` is a flat string | Large: no structured steps |
| `message.tool_calls` with args/result/time | No tool calls emitted | Large: tools not registered |
| `ModelMessage.metrics` token fields | Present in `RunCompleted` and `ModelRequestCompleted` | None |

---

## 6. Recommended Runtime Changes for v0.5.1

The original v0.5.1 plan assumed tool-call and reasoning events were already
available. This inspection shows they are not. The Runtime must first fix the
agent construction, then re-inspect the stream.

1. **Restore tool registration** in `build_recipe_agent()`:
   - Import and register `build_search_recipes_tool()` and
     `build_get_recipe_detail_tool()`.
   - Update instructions to refer to "tools" instead of "knowledge base".

2. **Add reasoning transparency**:
   - Add `ReasoningTools(add_instructions=True)` to the tool list, or
   - Decide with the Web team whether to render the flat `reasoning_content`
     string as-is.

3. **Re-run this inspection** after the agent changes to capture the new event
   shapes for `ToolCallStarted` / `ToolCallCompleted`.

4. **Token metrics** need no changes.
