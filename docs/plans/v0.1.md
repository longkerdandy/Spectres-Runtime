# v0.1 ŌĆö Development Environment Setup

**Goal:** Establish the engineering baseline for the Spectres-Runtime repository so that all subsequent feature work happens in a uniform, automated, reproducible environment. This release intentionally ships **no business functionality** (no AgentOS code, no agents, no database) ŌĆö only an "empty but engineering-complete" project skeleton.

**Definition of Done:**

- Any contributor can `git clone` the repo and, by following `README.md`, bring up a runnable empty application and pass all checks (`lint` / `type` / `test`) within 10 minutes.
- On every branch push and pull request, GitHub Actions runs the same checks, and code-scanning results appear in the GitHub Security tab.
- Every commit conforms to Conventional Commits; every branch and release action follows Git Flow.

## 1. Python Environment & Dependency Management

**Status: Ō£ģ Completed.**

- **Python version:** pinned to `3.12` (uv-managed `cpython-3.12.13`). System Python is left untouched.
  - `.python-version` contains `3.12`.
  - `pyproject.toml` declares `requires-python = ">=3.12,<3.13"` (upper bound locked to keep wheel availability predictable until 3.13 ecosystem matures).
- **Package manager:** [`uv`](https://docs.astral.sh/uv/) (v0.11.17+) ŌĆö fast, native `pyproject.toml` + lockfile support, compatible with the AgentOS ecosystem.
  - Project bootstrapped with `uv init --lib --build-backend hatch` (official template, `src/` layout, `py.typed` marker included).
  - Artifacts committed: `pyproject.toml`, `uv.lock`, `.python-version`.
- **Build backend:** `hatchling` (chosen by `uv init`'s `hatch` backend option; widely used, low-friction).
- **License declared via PEP 639 SPDX:** `license = "Apache-2.0"` (the `LICENSE` file itself ships in ┬¦10).
- **Runtime dependencies (`[project.dependencies]`) ŌĆö placeholders, no wiring yet:**
  - `agno>=2.6.9` ŌĆö will host AgentOS.
  - `fastapi>=0.136.3` ŌĆö web entry point.
  - `uvicorn[standard]>=0.48.0` ŌĆö ASGI server.
- **Dependency groups (PEP 735 `[dependency-groups]`):**
  - `lint` ŌĆö `ruff` only. Kept minimal so the CI lint job stays sub-second.
  - `test` ŌĆö `pytest`, `pytest-asyncio`, `pytest-cov`.
  - `dev` ŌĆö `mypy`, `pre-commit`, plus `include-group = "lint"` + `include-group = "test"`. This is uv's default group, so `uv sync` (no flags) installs the full developer environment in one command.
- **Why three groups, not one:** lets CI install only what each job needs (`uv sync --only-group lint` for the lint job, `--only-group test` for tests), keeps production images slim (`uv sync --no-dev`), and gives each group a clear semantic role. See `docs/plans/v0.1.md` design discussion.
- **Why `ruff` alone in `lint`:** ruff already subsumes flake8 + isort + black + pyupgrade + bugbear + simplify + a baseline of bandit-style security rules (`S`). Deep security scanning is handled by CodeQL (┬¦8); CVE scanning by Dependabot (┬¦8). Additional lint tools (codespell / vulture / interrogate) can be added incrementally if a concrete need appears.
- **Standard developer commands (to be documented in `README.md` under ┬¦10):**

  ```bash
  uv python install 3.12        # one-time, if uv hasn't already
  uv sync                       # installs runtime + dev (= lint + test + mypy + pre-commit)
  uv run python -c "import spectres_runtime; print(spectres_runtime.hello())"
  ```

- **CI-oriented commands (to be wired in ┬¦7):**

  ```bash
  uv sync --frozen --only-group lint     # lint job
  uv sync --frozen --only-group test     # test job
  uv sync --frozen --no-dev              # production / container image
  ```

- **Deliverables (all present):**
  - `.python-version`, `pyproject.toml`, `uv.lock`.
  - `src/spectres_runtime/__init__.py` + `src/spectres_runtime/py.typed` (placeholder package; real modules in ┬¦2).
  - `.venv/` reproducible via `uv sync --frozen` (excluded by `.gitignore`).

## 2. Empty Project Skeleton

**Status: Ō£ģ Completed.**

Minimal runnable structure, to be extended by later releases:

```
src/
ŌööŌöĆŌöĆ spectres_runtime/
    Ōö£ŌöĆŌöĆ __init__.py     # package metadata (__version__)
    Ōö£ŌöĆŌöĆ py.typed        # PEP 561 marker (shipped by uv init --lib)
    ŌööŌöĆŌöĆ app.py          # FastAPI() instance + /healthz endpoint + main()
tests/
ŌööŌöĆŌöĆ test_healthz.py     # smoke test via fastapi.testclient.TestClient
```

- **Layout:** `src/` layout (from `uv init --lib`), prevents implicit imports and forces installation through the build backend.
- **FastAPI app:** module-level `app: Final[FastAPI]` so `uvicorn spectres_runtime.app:app` works directly. Title/version/description set from project metadata.
- **`/healthz` endpoint:** async, no I/O, returns `{"status": "ok"}`. Serves as both a skeleton self-check and a probe for CI / container health checks. Tagged `meta` so it's grouped separately from future business endpoints in the OpenAPI schema.
- **`main()` callable:** kept in `app.py` for a future `[project.scripts]` entry point. **Not** registered in `pyproject.toml` for v0.1 ŌĆö `uvicorn spectres_runtime.app:app` is the standard launch command; scripts entry can be added later without breaking anything.
- **Smoke test:** uses `fastapi.testclient.TestClient` (synchronous wrapper, no event loop boilerplate). One test asserts status `200` and exact JSON payload. Verified passing with `uv run pytest`.
- **Known warning (non-blocking):** Starlette's `TestClient` emits `StarletteDeprecationWarning: Using httpx with starlette.testclient is deprecated; install httpx2 instead`. Test still passes; `httpx2` is too new to adopt. Suppressed in ┬¦3 via pytest `filterwarnings`.
- **Launch verified:** `uv run uvicorn spectres_runtime.app:app` boots successfully; `GET /healthz` returns `{"status": "ok"}`; `/openapi.json` auto-generated with the single endpoint listed.

## 3. Code Quality Tool Configuration

**Status: Ō£ģ Completed.**

All configuration lives in `pyproject.toml` under `[tool.*]`; no scattered `setup.cfg` / `.flake8` / `mypy.ini`. All three tools verified passing after install (`uv run ruff check . && uv run ruff format --check . && uv run mypy && uv run pytest`).

- **Ruff** (`[tool.ruff]` + `[tool.ruff.lint]` + `[tool.ruff.format]`)
  - `line-length = 100`, `target-version = "py312"`.
  - Rule sets enabled: `E, W, F, I, B, UP, SIM, RUF` ŌĆö the consensus modern-Python baseline used by FastAPI / httpx / pydantic. Kept intentionally small; expand only when a concrete need appears.
  - `per-file-ignores`: `tests/**/*.py` is allowed `S101` (`assert`) since pytest's model is built on it.
  - Formatter aligned with Black defaults: double quotes, spaces, LF ŌĆö explicit choices keep cross-platform diffs stable.
- **Mypy** (`[tool.mypy]`)
  - `python_version = "3.12"`, `strict = true` (bundles ~10 individual checks).
  - `files = ["src", "tests"]` ŌĆö type-checks both production code and tests.
  - `show_error_codes = true`, `pretty = true` ŌĆö error codes enable surgical `# type: ignore[code]` instead of blanket ignores.
  - Extra warn flags on: `warn_unused_ignores`, `warn_redundant_casts`, `warn_return_any`.
  - Override for `tests.*`: `disallow_untyped_decorators = false` so untyped third-party pytest decorators don't require stubs.
- **Pytest** (`[tool.pytest.ini_options]`)
  - `minversion = "9.0"`, `testpaths = ["tests"]`.
  - `addopts`: `-ra`, `--strict-markers`, `--strict-config`, `--cov=spectres_runtime`, `--cov-report=term-missing`, `--cov-fail-under=80`.
  - `filterwarnings`: suppresses Starlette's environmental `StarletteDeprecationWarning` (httpx ŌåÆ httpx2 nudge).
  - `asyncio_mode = "auto"` ŌĆö `async def test_*` works without per-test markers.
- **Coverage.py** (`[tool.coverage.run]` + `[tool.coverage.report]`)
  - `source = ["src/spectres_runtime"]`, `branch = true` ŌĆö both edges of conditionals must be exercised.
  - `exclude_lines`: `pragma: no cover`, `if __name__ == "__main__":`, `raise NotImplementedError`.
  - `show_missing = true`, `skip_covered = false` so all files appear in the report.
- **Coverage gate met:** baseline 100% (`__init__.py` + `app.py`); `main()` excluded via `# pragma: no cover` as a thin uvicorn wrapper exercised manually, not in unit tests.
- **Comment hygiene:** `pyproject.toml` comments explain *why* and *what* (decisions, semantics); status-shaped comments (`yet` / `for now` / `will be` / `once`) are avoided to prevent rot as the project evolves.

## 6. Git Flow Convention

Classic Git Flow, simplified ŌĆö documented in `docs/contributing.md` (created alongside this release):

| Branch | Purpose | From | Merges into |
|---|---|---|---|
| `main` | Released, stable, tagged | ŌĆö | ŌĆö |
| `develop` | Integration branch; the working surface of the next release | `main` | `main` (via release) |
| `feature/<name>` | New feature | `develop` | `develop` |
| `fix/<name>` | Non-urgent bug fix | `develop` | `develop` |
| `release/<version>` | Release prep ŌĆö version bump / changelog / minor fixes only | `develop` | `main` + `develop` |
| `hotfix/<name>` | Urgent production fix | `main` | `main` + `develop` |

- **Protection rules (GitHub Settings, documented):** `main` and `develop` reject direct pushes; PR + Ōēź 1 review + all-green CI required.
- **Tags:** semantic versioning `vMAJOR.MINOR.PATCH`. This release ships as `v0.1.0`.
